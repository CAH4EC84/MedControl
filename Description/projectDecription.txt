Пример интерфейса основной страницы доступной пользователю - https://ninjamock.com/Designer/Workplace/28836163/MainPage

Страница браузера содержит три основных компонента, формируемых в соответсвии с навигацией пользователя.

Верхняя часть разделена  между РОДИТЕЛЬСКОЙ таблицей данной страницы, навигации и формай редактирования данных.

Нижняя часть содержит табы с дополнительной информацией (ДОЧЕРНИЕ таблицы), открываемые по двойному клику на запись основной таблицы. (передается параметр :parentId)

Форма редактирования данных меняется в зависимости от того какая часть активна (верхняя \ нижняя).


Таблицы \ формы редактирования формируется унифицированной функцией на основании полученного объекта.
Стобцы РОДИТЕЛЬСКОЙ  таблицы имеют возможность поиска Like %%, возможность скрытия \ отображения столбцов.
Стобцы ДОЧЕРНЕЙ  таблицы имеют возможность поиска Like в пределах ранее полученных данных, возможность скрытия \ отображения столбцов.


Объекты получаемые из БД могут быть 2х типов
1) Родительская таблица, основная таблица на странице - фильтрация выполняется через обращение к БД.
2) Дочерняя таблица - открывается при клике на родительскую таблицу, связанна с ней через метку :parentTableId, занесенную в ID таблицы
    Фильтрация выполняется по полученными данным без запроса БД.

Так же деление таблиц как радительских так и дочерних выполняется на основании сущностей :
1) Объекты связанные с Hibernate  (оригинальные таблицы \ представления из бд)
2) Динамические объекты состоящите из объединенных таблиц (через JOIN's).

Интерфейс  dynBaseObject
1) getObjectType - тип объекта.  simple - объект имеет прямое отображение в БД (таблица \ представление). complex - объект полученный через JOIN.
2) getHql - запрос получающий все объекты данного типа из БД.
    для сложны таблиц получемых через JOIN  вызывает подметоды формируеющие отдельные части запроса (getSelect \ Join \ Where \ Group \ OrderPart).
3) visibleColumnsOrder - список \ размер столбцов отображаемых таблицой по умолчанию.
